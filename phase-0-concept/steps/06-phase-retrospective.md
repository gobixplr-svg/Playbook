# Step 6: Phase Retrospective

## Purpose

Capture what worked, what didn't, and what to improve for next time. This step closes Phase 0 and ensures process learnings are preserved — not just project deliverables.

## When to Run

After the Go/No-Go decision is made and all Phase 0 deliverables are finalized.

## Process

### 1. Review Deliverables Produced

List everything created during Phase 0. Confirm nothing is missing:

- [ ] Vision Statement
- [ ] Discovery Questionnaire (completed)
- [ ] Competitor Matrix + Deep-Dive Analyses
- [ ] Technical Feasibility Assessment
- [ ] Concept Document (with Go/No-Go decision)
- [ ] Concept Brief (one-page summary)
- [ ] ROADMAP.md (task tracking initialized)
- [ ] Project CLAUDE.md (AI context file)

### 2. Evaluate the Process

Ask these questions:

**What went well?**
- Which steps produced the most useful output?
- Where did the conversation flow naturally?
- What decisions were you most confident in?

**What was difficult?**
- Which questions were hardest to answer? Why?
- Where did research take longer than expected?
- Were any steps redundant or out of order?

**What was missing?**
- Did you discover gaps that the process didn't cover?
- Were there deliverables you wished you'd created earlier?
- Did context get lost between sessions?

**What would you change?**
- Should any steps be combined or reordered?
- Are there new steps that should be added?
- Should any templates be expanded or simplified?

### 3. Document Process Improvements

Record specific, actionable improvements. These feed back into the Playbook:

```markdown
## Process Improvements

### Add to Playbook
- [Specific change to a step or template]

### Change in Process
- [Order change, new checkpoint, etc.]

### New Template/Step
- [Something that should exist but doesn't]
```

### 4. Create Session Continuity Artifacts

If the concept phase spans multiple sessions (with an AI collaborator or team), ensure these artifacts exist for seamless pickup:

- **Concept Brief** — One-page summary of all decisions. Read this first in any new session.
- **Project CLAUDE.md** — AI context file with key decisions, tech stack, current phase, and "do not revisit" items.
- **ROADMAP.md** — Task tracking with next phase work broken into actionable items.

These three files are the "handoff package" — anyone (human or AI) should be able to read them and understand the project state in under 5 minutes.

### 5. Set Up Phase 1

Before closing Phase 0:

- [ ] ROADMAP.md has Phase 1 tasks defined
- [ ] Any spikes or research items from Phase 0 are captured as Phase 1 tasks
- [ ] The project directory structure is ready for Phase 1 work
- [ ] Git repository is initialized and committed (if not already)

## Key Principle

> The retrospective isn't about being thorough — it's about being honest. A two-line note about what actually needs to change is worth more than a page of vague observations.

## Checklist

- [ ] All Phase 0 deliverables confirmed complete
- [ ] Process reviewed (what worked / what didn't / what's missing)
- [ ] Improvements documented and fed back to Playbook
- [ ] Session continuity artifacts created (Brief, CLAUDE.md, ROADMAP.md)
- [ ] Phase 1 tasks initialized in ROADMAP.md
- [ ] Everything committed to version control
