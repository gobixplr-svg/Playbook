# Step 6: App Naming

## Purpose

Develop a shortlist of strong app name candidates and select a working name. This step uses the positioning statement, audience, differentiators, and competitive landscape from earlier steps to generate names that are distinctive, available, and aligned with the brand.

## When to Run

After the Go/No-Go decision (Step 5). You need the full concept context — audience, tone, differentiators, competitive names — before naming productively. Don't name something you might kill.

## Why a Process Matters

App naming feels like it should be intuitive, but "just pick something" leads to:
- Names that are already taken (App Store, domain, social handles)
- Names that sound like a competitor
- Names that don't convey the right feeling or category
- Attachment to a bad name because it was the first one you thought of

A structured process generates more options and filters them objectively.

## Process

### 6.1 Define Naming Criteria

Before generating names, define what a good name looks like for *this* app. Answer:

1. **Tone:** What should the name feel like? (e.g., adventurous, playful, premium, approachable, epic)
2. **Category signal:** Should the name hint at the app category? (e.g., "trail" or "journey" in a fitness app) Or be abstract?
3. **Length:** Preference for short (1 word), medium (2 words), or open?
4. **Audience fit:** Would your primary user be drawn to this name? Would it feel right on their home screen?
5. **Avoid:** Any words, patterns, or competitor names to steer clear of?

### 6.2 Generate Name Candidates

Use multiple approaches to generate a wide pool (aim for 30-50 raw candidates):

**Approach 1: Word Association**
Start with 5-10 core concept words (e.g., journey, trail, mile, quest, wander) and branch:
- Synonyms and related words
- Metaphors and analogies
- Compound words and mashups
- Verbs vs. nouns (action-oriented vs. identity-oriented)

**Approach 2: Positioning-Driven**
Pull from the positioning statement and core values:
- What's the one-word version of your differentiator?
- What emotion does the app create?
- What would a user say when describing it to a friend?

**Approach 3: Category Scan**
Look at how competitors and adjacent apps are named:
- What naming patterns dominate the category? (e.g., "[Activity] + [Goal]")
- Where can you break the pattern to stand out?
- What names are already taken that you need to avoid?

**Approach 4: Wordplay & Invention**
- Portmanteaus (combining two words)
- Modified spellings
- Foreign language words that sound good in English
- Acronyms or initialisms
- Made-up words that feel right phonetically

### 6.3 First Cut — Filter to 10-15

Remove candidates that fail any of these:

| Filter | Question |
|--------|----------|
| **Speakable** | Can you say it out loud without explaining the spelling? |
| **Memorable** | Would someone remember it after hearing it once? |
| **Tone match** | Does it match the tone criteria from 6.1? |
| **Not confusing** | Does it avoid unintended meanings or associations? |
| **Distinct** | Does it sound different from major competitors? |

### 6.4 Availability Check — Filter to 5-8

For each remaining candidate, check:

- [ ] **App Store:** Search iOS and Google Play. Is the name taken or too similar to an existing app?
- [ ] **Domain:** Is a reasonable domain available? (.com, .app, .io, or similar)
- [ ] **Social handles:** Check Instagram, TikTok, X/Twitter for handle availability
- [ ] **Trademark:** Quick search on USPTO TESS (US) or equivalent. Any registered marks in the same class?

Don't require perfection — a slight variation might work (e.g., "get[name].app" or "[name]hq" for handles). But if .com, the App Store name, and all social handles are taken, move on.

### 6.5 Stress Test — Narrow to 2-3

Put finalists through real-world scenarios:

1. **The friend test:** "Hey, I've been using this app called [name] — it tracks your walks as progress on real trails." Does it land?
2. **The search test:** If someone heard the name, would they find the right app by searching for it?
3. **The home screen test:** Picture the name under an icon on a phone. Does it look right? Too long? Too generic?
4. **The competitor test:** Put it next to competitor names. Does it stand out or blend in?
5. **The growth test:** Does the name still work if the app expands beyond v1? (e.g., would a "walk"-specific name limit you when you add cycling and rowing?)

### 6.6 Select a Working Name

Pick one. It doesn't have to be permanent — many successful apps have renamed (Instagram was Burbn, Slack was Tiny Speck). But having a working name:
- Unblocks Xcode project setup, Supabase config, and domain registration
- Creates emotional investment and project identity
- Stops you from using a placeholder that accidentally sticks

Document the decision:
```markdown
## Name Decision

**Selected:** [Name]
**Runners-up:** [Name 2], [Name 3]
**Rationale:** [Why this one won]
**Availability:** App Store ✓ | Domain ✓ | Social ✓ | Trademark ✓
```

## Deliverable

A naming decision documented in the Concept Document or as a standalone note in `docs/concept/`. Update the Concept Document, BRIEF.md, and CLAUDE.md to reflect the chosen name.

## Definition of Done

- [ ] Naming criteria defined (tone, length, category signal, audience fit)
- [ ] 30+ raw candidates generated from multiple approaches
- [ ] Filtered to 5-8 availability-checked finalists
- [ ] Stress-tested to 2-3 strong candidates
- [ ] Working name selected with rationale documented
- [ ] Availability confirmed (App Store, domain, social, trademark)
- [ ] Project documents updated with the chosen name

## Tips

- **Don't name by committee endlessly.** Set a timebox (1-2 hours for generation, 1 day for availability checks). Analysis paralysis on naming is real.
- **Say it out loud.** Names that look good on paper can sound awkward spoken. Test with real sentences.
- **Check in other languages.** If targeting international markets, make sure the name doesn't mean something unfortunate in major languages.
- **Secure what you can early.** Once you pick a name, register the domain and create social accounts immediately — even if you don't use them yet.
- **It's okay to rename later.** A working title gets the project moving. Don't let the perfect name block progress.
