## How Sprint 0 Differs from Feature Sprints

Sprint 0 is **spike-driven, not story-driven**. Instead of user stories with vertical slices (Data → Service → View → Tests), each item is a time-boxed experiment that answers a specific technical question. The output is knowledge and validated patterns, not shippable features.

**Use Sprint 0 when:**
- The project has 2+ "Complex" or "Unknown" items from Phase 0 feasibility
- The tech stack includes unfamiliar tools, engines, or SDKs
- Core mechanics need validation before building features on top of them
- The architecture depends on assumptions that haven't been tested in the target environment

**Skip Sprint 0 when:**
- All spikes were completed in Phase 1 Step 2
- The tech stack is fully familiar with production track record
- No architectural assumptions remain unvalidated
